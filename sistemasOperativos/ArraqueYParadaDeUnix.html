Arraque y Parada de UNIX

Introducción

En UNIX puede ser una tarea compleja el apagar o encender el S.O. Esta es una de las tareas del administrador, el cual debe conocer el proceso ya que el S.O. es especialmente vulnerable durante el arraque. Antes de apagar la máquina es necesario detener el sistema. Además puede haber varias razones por las cuales el adminsitrador quiera detener y arrancar el sistema en modo monousuario, como por ejemplo para tener el máximo control sobre el sistema. Algunas de las razones por las que el administrador pueda encender o apagar en monousuario son:

  • Añadir nuevo hardware al sistema.
  • Actualizar una aplicación.
  • El sistema de ficheros está corrupto.
  • Un hardware falla.
  • Realizar una copia de seguridad del sistema.

Niveles de Ejecución

UNIX tiene varios niveles de ejecución para poder limitar la actividad en el sistema mientras se hacen tareas administrativas. Podemos decir que un "nivel de ejecución" es un método que configura el sistema, indicando qué procesos ejecutar, si el modo es monousuario o multiusuario, qué dispositivos, etc. Hay 7 niveles de ejecución:

Nivel Función Descripción
0 Parada
  • Finaliza los servicios y los programas.
  • Desmonta el sistema de archivos.
  • Para la CPU.
1 Monousuario
  • Finaliza la mayoria de servicios.
  • Permite solo la entrada al administrador "root".
  • Utilizado para las tareas de mantenimiento y corregir errores críticos.
2 Multiusuario sin red.
  • No se inician los servicios de red.
  • Solo se permiten las entradas locales al sistema.
3 Multiusuario
  • Se inician todos los servicios menos los gráficos de X Window (Intefaz gráfica de UNIX).
4 Multiusuario
  • Igual que el 3.
  • Usado para hacer pruebas.
5 Multiusuario X
  • Inicia todos los servicios (= que el nivel 3).
  • Permite el login gráfico de los usuarios con Window X
6 Reinicio
  • Para todos los programas y servicios.
  • Reinicia el sistema.
Utilizamos el nivel 1 para realizar tareas administrativas ya que solo permite la entrada de root. Los niveles 2-5 permiten ejecutar el sistema con diferentes configuraciones, pudiendo iniciar (o no) X Window, elegir que dispositivos hardware iniciar, etc.

Arranque del Sistema

Al iniciar el sistema desde el estado de parada o pagado se denomina "booting" o "bootstrapping". Durante la fase de iniciar el sistema se carga en memoria un código denominado "boostrap loader" que se encarga de iniciar el S.O. Este trozo de código podemos almacenarlo:

  • En un disquete.
  • En disco duro.
  • En el MBR (Master Boot Record).
