Squash

Se utiliza para comprimir varios commits en uno solo. A veces tenemos una rama y estamos haciendo varios commits a esa rama, pero cuando nosotros queramos fusionarla con develop por ejemplo solo queremos hacer un solo commit con todos estos cambios. Esto es lo que se conoce como "squash". A continuación se muestra un ejemplo y como ser haría:

Imaginemos que partimos de la siguiente situación:

Nos ponemos en la rama y ponemos el siguiente comando: git rebase -i HEAD~3. Con este comando indicamos que queremos hacer un "rebase interactivo" desde HEAD (que se encuentar en el commit 5) hasta 3 commits para atrás.

Se nos abrirá el editor de texto que tengamos por defecto configurado en git (mirar la configuración de git). y nos saldrá los 3 últimos commits:

Tenemos que usar unos de los commits para hacer un commit que comprima todo, este commit lo marcaremos con "r" y pondremos un nuevo mensaje para este commit que tiene ya los otros commits comprimidos. Los commit que se incluyen dentro del otro lo marcaremos con la letra "f".
