El sistema operativo UNIX.

Introducción

Una de las tareas más importantes de un S.O. es su administración. Los errores procedentes de una mala administración del S.O. resultará en un funcionamiento inestable del S.O. y estos errores serán dificil de arreglar más tarde. Para poder administrar correctamente el S.O. se necesita tener un conocimiento profundo del S.O. y del hardware sobre el cual se está ejecutando. Normalmente los fabricantes de software publican unos controladores y herramientas para facilitar su administración, pero el problema es que muchos de estos hardwares no son compatibles unos con otros.

Administración de UNIX

La administración es el mantenimiento diario que permite que el sistema pueda crecer y se pueda modificar.

Responsabilidades del administrador

Por tanto el administrador es otro usuario que se encarga de mantener el correcto funcionamiento del S.O. En UNIX hay un id de presentación especial llamado "root", también denominado como "superusuario". Este tiene su propia contraseña (que se establecerá durante la instalación) y también tiene permisos especiales, como el acceso completo a todos los ficheros del sistema (y por tanto la capacidad de borrar todo). Entre las tareas de administración tenemos:

  • Alta y baja de identificadores de presentación de usuarios.
  • Instalación de nuevo software.
  • Modificación de ficheros de registro.
  • Hacer copias de seguridad.
  • Instalación de parches del S.O.
  • Actualizar paquetes del S.O. para arreglar errores funcionales o vulnerabilidades de seguridad.
Otras funciones del administrador del sistema son:
  • Arrancar y apagar el sistema: hay 7 niveles de ejecución.
    • Nivel 0: Parada. halt(detiene el sistema) y apagado.
    • Nivel 1: monousuario.
    • Nivel 2: multiusuario sin red.
    • Nivel 3: multiusuario sin gráficos X Window
    • Nivel 4: igual que el 3.
    • Nivel 5: multiusuario con X Window
    • Nivel 6: reinicio
  • Administrar la seguridad:
    • Controlar el acceso de los usuarios.
    • Gestionar las cuentas de los usuarios.
    • Gestionar los permisos de los usuarios.
  • Gestionar el espacio físico que el sistema monta durante el arranque.
    • Las particiones.
    • Unidades de disco.
    • Backup
  • Mantenimiento de archivos:
    • Backups.
    • Chequeo de discos.
  • Monitorización del sistema:
    • Comprobar el uso de:
      • La CPU.
      • Los discos.
      • La memoria.
    • Gestionar:
      • Las entradas/salidas.
      • La red
    • Medir el throughput(volumen de trabajo).
  • Identificar las causas de los problemas y solucinarlos.

Instalación

A diferencia de Windows, la instalación de Linux no es una tarea sencilla ya que nos permite un mayor control y personalización y esto resulta en que hay que tocar muchas más parámetros y opciones. Hay unas instrucciones detalladas en la iso o en el cd de instalación. Podemos encontrar la información de instalación de 2 formas: en los "HOWTO" y en los manuales desarrollados para la distribución. Además las distribuciones incluyen un manual de instalación de Linux en HTML o en pdf, dónde se explica con todo detalle que hay que saber a la hora de instalar el S.O.

Hay que tener muy en cuenta el concepto de partición. Cada S.O. organiza la información (los ficheros) de forma distinta, cada uno tiene tiene su propio sistema de archivos. Por esta razón no se pueden tener instalados a la vez varios S.O. mezclados en un mismo disco duro, para solucionar este problema surgieron las "particiones", con las que podemos dividir el disco duro y con esto conseguir tener varios sistemas operativos (podríamos tener un S.O. por partición, cada uno con su sistema de archivos).

Linux necesita dos particiones para poder funcionar, estas particiones se crearán durante la instalación:

  • Particion para el sistema de archivos en /.
  • Partición para el área de intercambio o Swap.
